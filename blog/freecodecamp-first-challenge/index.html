Freecodecamp challenge: Basic Javascript I

By Yann Hulot April 04, 2016 Comment Tweet Like +1

Congratulations on tackling your first challenge!

After finishing the first part of Freecodecamp, a lot of people find themselves lost when they first start the basic Javascript challenges. If you have experienced this or are experiencing it at the moment, do not despair. It will pass.

Remember, you can do this!

First step

Read the description of the challenge. Many times, I would start a challenge and speed read through the description then halfway through the challenge, I would find out that I had missed a crucial piece of information. If you do not want to waste your time like me, read the challenge description thoroughly.

Read the docs

I know that reading multiples pages of documents about Javascript functions can be boring or/and overwhelming. Just like you, I use to not read them but I have changed and so will you. You may wonder why you should read the docs if you don’t understand half of it?

The reason is simple, as a developer , you will have to read and understand other people’s docs when you’ll be tasked to maintain legacy code. Therefore, you better get use to read documents and try to make sense of them as much as you can. It doesn’t matter if you do not understand everything, the important part is that it becomes a part of your workflow.

Remember Copy/Pasting from stack overflow without understanding will make you feel good in the short-term but you won’t learn a thing. Don’t be lazy!

Write pseudo-code

Before writing any actual code, it is good practice to write in pseudo-code aka English what the goal of this particular challenge is and what steps you must go through in order to achieve the desired outcome.

For example in this case:

  1. split a string into an array
  2. reverse the order of the array’s elements
  3. join or glue together the array’s elements
  4. return(print) the new string to the console

The Code

As you will see below I have called my function reverseString but you can call yours any name you want. Same goes for str. Think of str as a placeholder for the real string(word) that will be passed as a parameter to the function.

You will notice that String starts with a capital letter and reverse with a regular one. This is a naming convention called camelCase or camelBack in this case. You can read more about it here and should start using it as soon as possible in your challenges.

Final word

If you feel lost or desperate, talk to somebody who is going through the same thing as you. You can join the freecodecamp Gitter channels or talk to members of other coding groups.

In any case, do not give up.
